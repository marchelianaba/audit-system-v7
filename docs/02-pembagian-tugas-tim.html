1. Komposisi Tim & Profil

Tim kecil 3 orang dengan pembagian peran berdasarkan keahlian dan komitmen waktu. Penamaan generic — sesuaikan dengan nama anggota tim Anda.

L Lead / Auditor PIC Anda — Inspektorat II, latar belakang auditor + product management

Tanggung jawab utama

  • Validasi requirement dari sudut auditor; pastikan output sesuai dengan kebutuhan SOP Inspektorat II.
  • Komunikasi dengan auditor pilot (3 orang IR-2) dan stakeholder (Inspektur II).
  • Review hasil KKP/LHR yang dihasilkan v7 untuk akurasi substansi (bukan teknis kode).
  • Prioritas backlog harian: tentukan bug mana fix dulu berdasarkan dampak ke workflow auditor.
  • Dokumentasi user-facing: user manual, troubleshooting guide.
  • Reporting progress mingguan ke Inspektur II.

Tugas teknis (light)

  • Buat penugasan test setiap pagi untuk Dev 1 & 2 latihan.
  • Manual test workflow E2E sebagai end-user (auditor).
  • Bantu setup Vercel/Fly secrets bila perlu.
  • Tidak menulis kode — kecuali edit prompt agen (markdown).

Komitmen waktu

  • ~20 jam/minggu (paruh waktu, paralel dengan tugas reguler IR-2).
  • Hadir di sync harian 30 menit + sync mingguan 1 jam.

B Developer 1 — Backend & Agent SDK Anggota Tim 1 — mahir Python, familiar dengan API/database

Tanggung jawab utama

  • Pengembangan FastAPI backend: routes, models, schemas, tool wrappers V6.
  • Integrasi Claude Agent SDK: konfigurasi agen, prompt tuning, debugging tool call.
  • Database: migrasi, query optimasi, integritas audit trail.
  • Tool layer V6: wrapping run_batch.py, qc_saipi.py, render_*.py.
  • Deployment Fly.io: monitoring, secrets, scaling, log analysis.
  • Testing: smoke test script, agent simulation, V6 script validation.

Yang TIDAK dikerjakan

  • UI design / styling (itu Dev 2).
  • Substansi audit / interpretasi temuan (itu Lead).

Komitmen waktu

  • ~40 jam/minggu (full time).
  • Standby untuk hotfix saat pilot week.

F Developer 2 — Frontend & UX Anggota Tim 2 — mahir Next.js/React, TypeScript, Tailwind

Tanggung jawab utama

  • Pengembangan Next.js frontend: halaman, komponen, state management.
  • UX: loading states, error handling, empty states, toast notifications.
  • API integration: fetch wrapper, type safety, error parsing.
  • Deployment Vercel: env vars, custom domain (kalau ada), preview deployments.
  • UI polish: spacing, typography, accessibility minimal.
  • Testing: manual UI test, screenshot regression, cross-browser (Chrome + Firefox).

Yang TIDAK dikerjakan

  • Backend logic / agent prompts (itu Dev 1).
  • Format LHR/KKP Word (itu V6 renderer, dipanggil backend).

Komitmen waktu

  • ~40 jam/minggu (full time).
  • Standby pilot week untuk UI hotfix.
Catatan komitmen: Lead 20 jam + Dev 1 & 2 masing-masing 40 jam = ~100 jam-orang/minggu × 4 minggu = ~400 jam total. Buffer 20% untuk overflow → ~480 jam-orang. Realistis untuk scope prototype.

3. Matriks RACI Aktivitas Kunci

RACI = Responsible (mengerjakan), Accountable (bertanggung jawab final), Consulted (perlu dimintai pendapat), Informed (perlu tahu). Mencegah miskomunikasi siapa yang ambil keputusan.

Aktivitas / Keputusan Lead Dev 1 (BE) Dev 2 (FE)

Aturan praktis: Untuk setiap PR/perubahan, owner-nya satu orang (R). Yang lain di-tag sebagai reviewer (C) bila perlu. Lead selalu dapat update kalau ada perubahan yang impact ke user-facing.

4. Tugas Detail per Minggu

Breakdown tugas yang sesuai dengan roadmap 4 minggu. Setiap orang tahu apa yang dikerjakan setiap minggu.

Minggu 1 (19–25 Mei) — Bug-fix + E2E Pengadaan

Lead

  • Buat 2 penugasan test (Reviu Pengadaan) dengan data DC-DRC PSrE.
  • Jalankan workflow sebagai auditor; catat semua friction point ke issue tracker.
  • Review output KKP draft yang dihasilkan agen — apakah substansi masuk akal?
  • Edit prompt anggota_tim.md bila ada panduan reviu yang kurang.
  • Sync harian 30 menit, sync mingguan Jumat 1 jam.

Dev 1 — Backend

  • Fix bug yang muncul saat Lead test (priority 1).
  • Endpoint GET /output/{penugasan_id}/{filename} serve file dari folder penugasan.
  • Test integrasi qc_saipi.py V6 dipanggil setelah KKP.
  • Logging structured (JSON) untuk semua agent runs.
  • Setup error tracking (Sentry atau cukup Fly logs filtering).

Dev 2 — Frontend

  • Tab "Output & QC" dengan list file output + tombol Download.
  • Banner error yang jelas (bukan "Connection error").
  • Loading spinner saat agen berjalan dengan counter detik.
  • Toast sukses setelah action selesai.
  • Disable form submit saat request berjalan.

Minggu 2 (26 Mei–1 Juni) — E2E RKA-K/L + Ketua Tim

Lead

  • Buat 2 penugasan test Reviu RKA-K/L dari paket Dit Pengendalian 2027.
  • Jalankan workflow Ketua Tim — input form LHR (judul, auditi, dasar, dst.).
  • Bandingkan LHR draft v7 vs hasil V6 atau template — apakah konsisten?
  • Edit ketua_tim.md prompt bila narasi LHR perlu polish.
  • Tulis sample data fixture untuk testing (lihat dok testing).

Dev 1 — Backend

  • Test & fix pipeline reviu-rka-kl/run_batch.py integration.
  • Approval workflow: endpoint untuk update status sasaran & penugasan.
  • QC SAIPI stage LHP test & fix.
  • Optimasi: aktifkan prompt caching untuk skill content yang berulang.
  • Setup automated smoke test (pytest atau bash script).

Dev 2 — Frontend

  • Tab "Chat KT" dengan form input narasi LHR (judul, auditi, dst.).
  • Tombol "Setujui KKP" / "Setujui LHR" dengan konfirmasi modal.
  • Status badge yang jelas di setiap tahap (DRAFT, SELESAI_KKP, LHP_DONE).
  • Preview DOCX inline (gunakan iframe + Microsoft Office viewer atau download saja).
  • Cross-test di Chrome & Firefox.

Minggu 3 (2–8 Juni) — Polish + Multi-user

Lead

  • Bikin skenario multi-user: 2 AT + 1 KT kerjakan penugasan sama bersamaan.
  • Draft user manual versi awal (5 halaman) dari sudut auditor.
  • Kontak 3 auditor IR-2 untuk konfirmasi available di minggu 4 pilot.
  • Briefing internal IR-2 tentang prototype (15 menit di rapat reguler).

Dev 1 — Backend

  • Quota check per user (max 5 agent runs/jam).
  • Rate limiting di FastAPI middleware.
  • Backup database harian (cron job di Fly).
  • Security review: JWT expiry, role check audit, file upload validation.
  • Performance: cek slow query, tambah indeks DB bila perlu.

Dev 2 — Frontend

  • Tab "Riwayat" — tampilkan agent_runs dengan filter.
  • Modal detail tool calls untuk transparansi.
  • Polish konsistensi: spacing, typography, button states.
  • Empty states yang informatif (bukan halaman kosong).
  • Tooltip pada tombol disabled (kenapa belum aktif).

Minggu 4 (9–15 Juni) — Pilot + Dokumentasi

Lead

  • Senin: Briefing 30 menit dengan 3 auditor pilot.
  • Selasa–Kamis: Hadir penuh, kumpulkan feedback langsung, koordinasi fix prioritas.
  • Jumat: Debrief 1 jam dengan auditor; tulis ringkasan untuk Inspektur II.
  • Sabtu: Finalisasi user manual + admin guide.
  • Minggu: Slide presentasi (10 slide) untuk Inspektur II.

Dev 1 — Backend

  • Standby setiap hari selama pilot (Senin–Jumat).
  • Hotfix backend bug dalam < 2 jam dari laporan.
  • Monitoring biaya API harian; alert bila > USD 10/hari.
  • Backup database sebelum & sesudah pilot.
  • Tulis admin guide: redeploy, troubleshooting, rollback.

Dev 2 — Frontend

  • Standby setiap hari selama pilot.
  • Hotfix UI bug dalam < 2 jam dari laporan.
  • Bantu Lead screenshot fitur untuk user manual.
  • Final UI polish berdasarkan feedback auditor.
  • Cleanup repo: remove unused code, tambah comments.

5. Struktur Sync & Komunikasi

Cadence reguler untuk koordinasi tanpa terlalu sering meeting. Total ~5 jam meeting/minggu — sisanya kerja efektif.

Daily Stand-upSetiap hari 09.00
15 menit
Format: setiap orang jawab 3 hal — (1) kemarin saya kerjakan apa, (2) hari ini saya kerjakan apa, (3) ada blocker?

Tujuan: sinkronisasi cepat, deteksi blocker dini, hindari double-work.

Tools: Google Meet / Zoom + Notion log. Atau cukup chat di WA grup "v7 Daily" — kalau sehat dan tidak ada blocker, tulis text update saja.
Mid-week SyncRabu 14.00
30 menit
Format: review progress vs target minggu ini. Demo 5 menit dari setiap dev. Adjust prioritas sisa minggu.

Tujuan: early warning kalau target minggu ini tidak tercapai. Lead bisa intervensi/repriotitas.

Output: update di Notion progress board.
Weekly RetroJumat 15.30
1 jam
Format: apa yang work, apa yang tidak work, action items minggu depan. Demo akhir minggu — workflow end-to-end yang fix di minggu itu.

Tujuan: learning + planning. Setiap orang sampaikan satu hal yang bisa diperbaiki di tim.

Output: retro notes + backlog minggu depan terupdate.
Pilot DailyMinggu ke-4
17.00, 30 menit
Format: review bug yang dilaporkan auditor, prioritas hotfix. Lead share feedback verbal yang tidak masuk issue tracker.

Khusus minggu pilot — siklus iterasi 24 jam (laporan hari ini → fix besok pagi → deploy → test).

7. Eskalasi & Decision Making

Aturan kapan harus eskalasi dan kepada siapa.

SituasiEskalasi keDalam waktu
Bug yang block workflow E2E ditemukan saat developmentDaily stand-up esok hari< 24 jam
Bug yang block workflow saat pilotWA Lead langsung< 30 menit
Konflik teknis antara Dev 1 & Dev 2Lead untuk arbiter< 4 jam
Biaya API > USD 10/hari mendadakLead + matikan agent endpoints sementara< 1 jam
Fly/Vercel down > 30 menitLead untuk info ke pilot user, status update< 15 menit
Auditor pilot tidak puas / menyerahLead reach out personal + Inspektur IIHari yang sama
Anggota tim sakit/cuti > 2 hariLead reassign + sync ulang prioritas< 4 jam
