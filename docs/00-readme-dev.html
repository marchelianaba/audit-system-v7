1. Strategi Dual-Device

Aturan main supaya kerja di 2 device tidak berantakan.

Prinsip dasar

  1. Git adalah sumber kebenaran. Semua kode hanya di GitHub. Setiap device melakukan clone dari sana. Jangan pernah copy folder langsung antar device.
  2. Selalu push sebelum tutup. Sebelum tutup laptop di akhir hari, commit + push semua perubahan. Esok lanjut di device lain.
  3. Selalu pull sebelum mulai. Buka device → terminal pertama yang Anda jalankan adalah git pull.
  4. Venv & node_modules tidak ikut Git. Setiap device punya environment masing-masing — di-install lokal, tidak di-sync.
  5. .env file tidak ikut Git. Setiap device punya .env sendiri. Catat secret di password manager (1Password, Bitwarden), bukan di repo.

Pembagian kerja antar device

AktivitasDevice yang idealAlasan
Coding intensif (refactor, bug fix kompleks)Yang punya monitor besarMulti-window dev nyaman
Test workflow E2E sebagai auditorBebas, asal browser OKHanya butuh browser + terminal SSH
Demo ke stakeholderMacBook (lebih portable)Mudah bawa ke ruang rapat
Deploy ke Fly/VercelBebas, asal CLI ter-install—
Long-running task (build, test)Device yang ditinggal—
Hindari: coding di 2 device sekaligus dalam waktu sama tanpa coordinate. Bisa terjadi merge conflict kalau lupa pull. Aturan: 1 device aktif coding per saat.

3. Setup Pertama di Windows

Skip langkah yang sudah pernah Anda lakukan. Folder project diasumsikan di C:\Users\irfan\Downloads\audit-system-v7\.

1
Install tools yang belum ada

Buka PowerShell as Administrator (kalau perlu install Chocolatey untuk batch install):

Set-ExecutionPolicy Bypass -Scope Process -Force
iex ((New-Object System.Net.WebClient).DownloadString('https://chocolatey.org/install.ps1'))

# Install tools sekaligus
choco install -y python312 nodejs-lts docker-desktop git vscode flyctl

# Vercel via npm
npm install -g vercel

Atau install manual: Python dari python.org, Node dari nodejs.org, Docker Desktop, Git, VS Code, flyctl dari fly.io/docs.

2
Clone repo dari GitHub 
cd C:\Users\irfan\Downloads
git clone https://github.com/[username]/audit-system-v7.git
cd audit-system-v7

(Ganti URL dengan repo Anda. Bila belum di GitHub, push dulu dari folder current — lihat §6.)

Setup .env 
cp .env.example .env
# Edit .env, isi ANTHROPIC_API_KEY dan APP_SECRET_KEY
notepad .env

Setup backend venv (DI LUAR folder project)

Pelajaran sebelumnya: venv di Dropbox bermasalah. Kalau Anda di Downloads (bukan Dropbox), venv di folder project OK. Tapi pola amannya tetap di luar:

Init database + seed users

Login Fly & Vercel

4. Setup Pertama di macOS

Asumsi MacBook baru tanpa dev tools. Pakai Homebrew untuk install cepat semuanya.

1
Install Homebrew

Buka Terminal (Cmd+Space → ketik "Terminal"):

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Setelah selesai, ikuti instruksi yang muncul untuk tambah brew ke PATH
# Biasanya:
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
2
Install semua tools via brew 
brew install python@3.12 node@20 git
brew install --cask docker visual-studio-code

# flyctl
brew install flyctl

# Vercel
npm install -g vercel

Setelah install Docker Desktop, buka aplikasinya sekali untuk inisialisasi (icon Docker harus muncul di menu bar).

3
Setup SSH key untuk GitHub (kalau belum) 
ssh-keygen -t ed25519 -C "email@komdigi.go.id"
# Enter terus untuk default

# Copy public key ke clipboard
pbcopy < ~/.ssh/id_ed25519.pub

# Buka GitHub → Settings → SSH and GPG keys → New SSH key → paste
4
Clone repo 
cd ~
mkdir -p Projects
cd Projects
git clone git@github.com:[username]/audit-system-v7.git
cd audit-system-v7
5
Setup .env 
cp .env.example .env
# Edit dengan editor pilihan
code .env
# Isi ANTHROPIC_API_KEY dan APP_SECRET_KEY (sama dengan Windows)
6
Setup backend venv 
# Buat di home dir, di luar project
mkdir -p ~/envs
python3.12 -m venv ~/envs/audit-v7

# Activate
source ~/envs/audit-v7/bin/activate

# Install deps
cd ~/Projects/audit-system-v7/backend
pip install --upgrade pip
pip install -r requirements.txt
7
Setup Postgres + init DB 
# Pastikan Docker Desktop jalan
cd ~/Projects/audit-system-v7
docker-compose up -d db
docker ps   # verifikasi

# Init DB
cd backend
python -m app.init_db
8
Setup frontend + Fly/Vercel login 
cd ~/Projects/audit-system-v7/frontend
npm install
cp .env.example .env.local

# Login Fly & Vercel
fly auth login
vercel login
Tip macOS: Pakai iTerm2 (brew install --cask iterm2) sebagai Terminal yang lebih powerful. Split pane Cmd+D untuk backend & frontend di satu window.

5. Workflow Harian (Mulai Kerja)

Rutinitas pagi dan sore yang harus konsisten supaya tidak kehilangan kode.

Pagi (mulai kerja)

W Windows PowerShell

# 1. Pull latest
cd C:\Users\irfan\Downloads\audit-system-v7
git pull

# 2. Pastikan Docker jalan, start DB
docker-compose up -d db

# 3. Backend (terminal 1)
C:\envs\audit-v7\Scripts\Activate.ps1
cd backend
pip install -r requirements.txt   # kalau ada update
uvicorn app.main:app --reload --port 8000

# 4. Frontend (terminal 2 — buka window baru)
cd C:\Users\irfan\Downloads\audit-system-v7\frontend
npm install                        # kalau ada update
npm run dev

macOS Terminal

# 1. Pull latest
cd ~/Projects/audit-system-v7
git pull

# 2. Pastikan Docker jalan, start DB
docker-compose up -d db

# 3. Backend (terminal 1)
source ~/envs/audit-v7/bin/activate
cd backend
pip install -r requirements.txt   # kalau ada update
uvicorn app.main:app --reload --port 8000

# 4. Frontend (terminal 2 — Cmd+T tab baru)
cd ~/Projects/audit-system-v7/frontend
npm install                        # kalau ada update
npm run dev

Buka browser ke http://localhost:3000. Edit kode → save → auto-reload.

Sore (sebelum tutup laptop)

  1. Stop uvicorn & npm dev (Ctrl+C di kedua terminal).
  2. Commit perubahan: 
    git status                    # cek apa yang berubah
git add .
git commit -m "feat: deskripsi singkat perubahan"
git push origin main
  3. Stop Docker container (opsional, hemat resource): 
    docker-compose down
  4. Tutup laptop. Esok di device lain, mulai dari "Pagi" lagi.
JANGAN PERNAH: tutup laptop dengan kode yang belum di-commit. Risiko terbesar dual-device: lupa push → besok di device lain modifikasi file yang sama → merge conflict besar.

Saat ganti device di tengah hari

  1. Device lama: git add . && git commit -m "wip: jeda istirahat" && git push (WIP = work in progress, OK untuk commit setengah jadi)
  2. Device baru: git pull
  3. Lanjut kerja.

Branch strategy untuk prototype

Untuk tim kecil 3 orang dan timeline pendek, branch strategy sederhana:

  • main — selalu siap deploy, otomatis Fly & Vercel deploy dari sini.
  • feature/<nama> — bila ada perubahan besar yang butuh review (mis. feature/audit-trail-tab). Merge ke main setelah review.
  • Untuk hot fix kecil, push langsung ke main OK.
Aturan tim: Tidak ada force push ke main (git push --force). Bila ada conflict, koordinasi via WA dulu sebelum merge.

7. Deployment

Sama persis untuk kedua device — tools (flyctl, vercel) lintas platform.

Deploy backend ke Fly.io

Dari kedua device, perintah identik:

W Windows

# Copy V6 scripts (sekali per device atau saat update)
cd C:\Users\irfan\Downloads\audit-system-v7
# (Lihat scripts/deploy-fly.ps1 - jalankan)
.\scripts\copy-v6-scripts.ps1

# Deploy
cd backend
fly deploy --app audit-ai-v7

# Cek hasil
fly status --app audit-ai-v7
fly logs --app audit-ai-v7 --no-tail | Select-Object -Last 30

macOS

# Copy V6 scripts
cd ~/Projects/audit-system-v7
bash scripts/deploy-fly.sh   # script bash sudah ada

# Atau manual:
cd backend
fly deploy --app audit-ai-v7

# Cek hasil
fly status --app audit-ai-v7
fly logs --app audit-ai-v7 --no-tail | tail -30

Deploy frontend ke Vercel

# Sama di Windows & macOS
cd [path-ke-]/audit-system-v7/frontend
vercel --prod

# URL production akan di-print di akhir output

Update secret di Fly

# Ganti / set ulang secret (bisa dari device manapun yang sudah login)
fly secrets set ANTHROPIC_API_KEY=sk-ant-new... --app audit-ai-v7

# List secrets (tidak menampilkan value, hanya nama)
fly secrets list --app audit-ai-v7

# Hapus secret
fly secrets unset SECRET_NAME --app audit-ai-v7

SSH ke Fly machine untuk debug

# Sama di kedua OS
fly ssh console --app audit-ai-v7

# Setelah masuk, navigasi:
cd /app
ls /v6/scripts/
ls /data/penugasan/

# Exit
exit
Konvensi tim: Setiap selesai deploy, tulis di WA grup format: 🚀 backend deploy main@abc1234 — fix CORS for SSE. Anggota lain tahu ada perubahan, bisa pull + test.
