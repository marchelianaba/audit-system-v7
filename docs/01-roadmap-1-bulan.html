2. Tujuan 1 Bulan

Definisi spesifik dari "selesai" untuk prototype ini.

Goal utama

  1. Workflow Reviu Pengadaan E2E lulus: AT upload KAK+HPS+RFI → KKP → QC PASS → KT setujui → LHR.docx siap.
  2. Workflow Reviu RKA-K/L E2E lulus: serupa dengan TOR+RAB sebagai input.
  3. Pilot terbatas berhasil: 3 auditor IR-2 jalankan masing-masing 1 penugasan riil, hasil bisa dipakai.
  4. Dokumentasi lengkap: user manual, troubleshooting guide, daftar known limitations.

Non-goal (sengaja dikecualikan)

  • CACM, scheduler otomatis, notifikasi WA/email — tahap-2.
  • Auto-inject ke INTEGRAL — upload manual untuk sekarang.
  • SSO Komdigi — masih login email+NIP.
  • Mobile responsive — desktop only.
  • Migrasi ke PDN — masih Fly.io Singapore.

9. Kriteria Sukses (Definition of Done)

Pernyataan tegas yang harus terpenuhi di akhir 1 bulan supaya prototype dianggap berhasil.

Mandatory (harus semuanya)

  • ✅ Reviu Pengadaan E2E selesai untuk min. 2 paket berbeda tanpa intervensi developer.
  • ✅ Reviu RKA-K/L E2E selesai untuk min. 2 paket berbeda tanpa intervensi developer.
  • ✅ QC SAIPI berjalan otomatis di akhir KKP & LHR, status PASS terdeteksi benar.
  • ✅ 3 auditor IR-2 menjalankan masing-masing 1 penugasan riil. Min. 2 dari 3 nyatakan output bisa dipakai.
  • ✅ Output LHR.docx bisa di-download, format sesuai template Inspektorat II.
  • ✅ Audit trail tampak di UI (siapa kerjakan apa, kapan).
  • ✅ User manual min. 10 halaman tersedia PDF.
  • ✅ Total biaya bulan ke-1 ≤ USD 200 (Rp 3,2 juta).

Nice-to-have (boleh skip kalau waktu mepet)

  • Auto-polling status dokumen di UI (sekarang refresh manual).
  • Toast notifikasi untuk event penting.
  • Tab Riwayat dengan visualisasi tool calls.
  • Export laporan QA ke PDF.
Definisi GAGAL: Bila di akhir minggu ke-4 tidak ada satupun penugasan riil yang berhasil selesai E2E di v7 (semua butuh fallback ke V6), atau biaya bulan ke-1 > USD 500, prototype dianggap belum siap pilot lebih luas dan harus diiterasi.
